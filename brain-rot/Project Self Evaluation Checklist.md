# Project Self Evaluation Check list *Make a copy of this document for yourself*

## ---

## **1\. Problem Fit** 

## **☐ Real pain, real people**  	Someone actually does this today (manually, badly, or expensively)?

## **☐ Repeated task** 	 Happens weekly or more (not once in a lifetime).

- [ ] **☐ Execution problem**  
       	The issue is *doing the work*, not just *knowing information*.

If the problem is mostly “people want better advice” → ❌

#### **2\. Is it Agentic?**

- [ ] Requires reasoning \+ decision-making? Not just summarizing or generating text.  
- [ ] Needs memory or context over time. User history, preferences, or past actions matter.  
- [ ] Has “what should I do next?” moments. The agent chooses actions, not just answers.

If a single prompt could solve it →  ❌

####  **3\. Is it Rule Based?**

Answer yes to at least 2 of these

- [ ] Inputs are messy or unstructured (text, voice, PDFs, screenshots)  
- [ ] Evolving rules/edge cases common?  
- [ ] Logic would explode if written as rules

If rules clearly work → ❌

#### **4\. MVP/MEP Scope (Can this be built in 6 weeks)**

- [ ] One persona only((No multi-sided platforms.)  
- [ ] One core job (Not planning \+ tracking \+ optimizing \+ reporting)  
- [ ] One agentic loop (Input → reason → act → feedback → improve)  
- [ ] Can demo value with fake, synthetic, open source or user-provided data (No massive integrations required)

If MVP needs 10 featuresand complex integrations  → ❌

#### **5\. Why not solve this ChatGPT?**

- [ ] Needs persistent memory or workflow  
- [ ] Needs structured outputs or tracking \- Not just chat.  
- [ ] Needs repeated use over time to value compounds.

If ChatGPT \+ prompt \= 80% solution → weak idea

### **One Rule to Remember**

###  In  this course iterations matter more than fancy products. A tiny agent that reasons well beats a big shiny app that doesn’t solve a real challenge.
